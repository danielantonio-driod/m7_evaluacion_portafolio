# Portafolio de Evaluación Django

Este portafolio contiene ejemplos prácticos de integración y uso de Django con bases de datos, cubriendo los principales conceptos y competencias técnicas adquiridas en el curso. Cada módulo aborda un aspecto fundamental del desarrollo con Django, desde la configuración inicial hasta la implementación de aplicaciones completas.

## Estructura de Módulos

### 01-integracion-django-bd
**Función:** Explica cómo Django se integra con diferentes sistemas de bases de datos (SQLite, PostgreSQL, MySQL), cómo se configura el archivo `settings.py` y cómo el ORM gestiona las conexiones y operaciones.

### 02-modelos-simples
**Función:** Implementa un modelo simple e independiente en Django. Ejemplo de la entidad `Producto` sin relaciones, mostrando cómo crear tablas básicas en la base de datos.

### 03-modelos-relaciones
**Función:** Implementa modelos con relaciones uno a uno, uno a muchos y muchos a muchos. Ejemplo con entidades `Cliente`, `Producto`, `Pedido` y `PerfilCliente` para mostrar la interconexión entre tablas usando el ORM de Django.

### 04-migraciones
**Función:** Proyecto de prueba para demostrar la creación y aplicación de migraciones en Django, permitiendo la propagación de cambios en el esquema de la base de datos.

### 05-consultas-orm-sql
**Función:** Proyecto funcional con app `pedidos` que muestra cómo realizar consultas de filtrado y personalizadas usando el ORM y sentencias SQL, recuperando información relevante de la base de datos.

### 06-app-mvc-crud
**Función:** Proyecto funcional con app `productos` que implementa operaciones CRUD (Crear, Leer, Actualizar, Eliminar) usando vistas, modelos y templates, siguiendo el patrón MVC de Django.

### 07-apps-preinstaladas
**Función:** Proyecto funcional con app `productos` para mostrar la utilidad de las aplicaciones preinstaladas de Django (`admin`, `auth`, `sessions`) y cómo personalizar el panel de administración para gestionar modelos.

---

## Resumen General
- Cada módulo es un proyecto Django independiente, con su propio entorno y estructura.
- Se cubren desde conceptos básicos de configuración y modelos, hasta ejemplos avanzados de relaciones, migraciones, consultas y aplicaciones CRUD.
- El portafolio demuestra el dominio de Django y su integración con bases de datos, así como el uso de sus herramientas y buenas prácticas.
